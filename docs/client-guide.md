## Установка

### Linux / macOS

Установка одной командой:

```bash
curl -fsSL https://fxtun.dev/install.sh | sh
```

Скрипт автоматически определит вашу ОС и архитектуру, скачает бинарник в `~/.local/bin/` и создаст симлинки `fxtunnel` и `fxtun`.

**Поддерживаемые платформы:**
- Linux: amd64, arm64
- macOS: amd64 (Intel), arm64 (Apple Silicon)

### Windows

PowerShell (запустить от администратора):

```powershell
irm https://fxtun.dev/install.ps1 | iex
```

Или скачайте `.exe` вручную со [страницы загрузок](https://fxtun.dev/downloads).

> **SmartScreen:** Windows может показать предупреждение «Windows защитила ваш компьютер» при первом запуске. Бинарники пока не подписаны сертификатом. Нажмите **«Подробнее»** → **«Выполнить в любом случае»**.

### GUI-клиент

Десктопное приложение с графическим интерфейсом доступно для Linux и Windows на [странице загрузок](https://fxtun.dev/downloads).

### Проверка установки

```bash
fxtunnel version
```

Вывод:
```
fxTunnel Client vX.Y.Z (built 2026-04-01T10:30:45Z)
GitHub: https://github.com/mephistofox/fxtun.dev
Website: https://fxtun.dev
```

## Аутентификация

Для работы с fxTunnel нужен API-токен. Получите его в [личном кабинете](https://fxtun.dev/login) → раздел «Токены».

### Сохранение токена

Интерактивный режим (предложит выбрать способ):

```bash
fxtunnel login
```

Варианты:
1. **Ввести токен** — вставьте API-токен вручную
2. **Через браузер** — откроется страница авторизации, токен сохранится автоматически

Или напрямую через флаг:

```bash
fxtunnel login -t sk_ваш_токен
```

Токен сохраняется в системном хранилище:
- **macOS:** Keychain
- **Linux:** Secret Service (GNOME Keyring / KDE Wallet)
- **Windows:** Credential Manager

### Удаление токена

```bash
fxtunnel logout
```

### Использование без сохранения

Можно передавать токен при каждом вызове через флаг `--token` или переменную окружения:

```bash
fxtunnel http 3000 --token sk_ваш_токен

# Или через env
export FXTUNNEL_SERVER_TOKEN=sk_ваш_токен
fxtunnel http 3000
```

## Быстрый старт

### HTTP-туннель

Откройте локальный веб-сервер на порту 3000:

```bash
fxtunnel http 3000
```

Вывод:
```
Tunnel established!
HTTP: https://ab1cd2ef.fxtun.dev
Forwarding to localhost:3000
Inspector: http://127.0.0.1:4040
```

Теперь `https://ab1cd2ef.fxtun.dev` доступен из интернета и проксирует трафик на ваш `localhost:3000`.

### TCP-туннель

Откройте SSH-сервер:

```bash
fxtunnel tcp 22
```

Вывод:
```
Tunnel established!
TCP: fxtun.dev:51234
Forwarding to localhost:22
```

Подключение: `ssh user@fxtun.dev -p 51234`

### UDP-туннель

Откройте локальный DNS-сервер:

```bash
fxtunnel udp 53
```

## HTTP-туннели

```bash
fxtunnel http <порт> [флаги]
```

### Свой поддомен

```bash
fxtunnel http 3000 --domain myapp
# → https://myapp.fxtun.dev
```

Флаги `--domain` и `--subdomain` — синонимы. Если поддомен не указан, генерируется автоматически.

### Basic Auth

Защитите туннель логином и паролем:

```bash
fxtunnel http 3000 --auth user:MySecurePass123
```

Требования: пароль минимум 8 символов, имя пользователя не должно содержать `:`.

Все запросы к туннелю потребуют HTTP Basic Auth:

```bash
curl -u user:MySecurePass123 https://myapp.fxtun.dev
```

### Ограничение по IP

Разрешите доступ только с определённых IP или подсетей:

```bash
fxtunnel http 3000 --allow-ip 203.0.113.10 --allow-ip 10.0.0.0/8
```

Флаг `--allow-ip` можно указывать несколько раз. Поддерживаются одиночные IP и CIDR-нотация.

### Авто-закрытие

Туннель автоматически закроется после указанного периода неактивности:

```bash
fxtunnel http 3000 --auto-close 30m
```

Допустимый диапазон: от `1m` до `24h`. Таймер сбрасывается при каждом запросе.

### Максимальное время жизни

Туннель принудительно закроется через указанное время, независимо от активности:

```bash
fxtunnel http 3000 --max-lifetime 8h
```

Допустимый диапазон: от `1m` до `7d`.

### Комбинирование флагов

```bash
fxtunnel http 3000 \
  --domain myapp \
  --auth admin:StrongPass123 \
  --allow-ip 10.0.0.0/8 \
  --auto-close 1h \
  --max-lifetime 24h
```

### Все флаги HTTP

| Флаг | Короткий | Описание | По умолчанию |
|------|----------|----------|--------------|
| `--domain` | `-d` | Поддомен | Авто |
| `--subdomain` | | Синоним --domain | Авто |
| `--auth` | | Basic Auth (user:password) | Нет |
| `--allow-ip` | | Разрешённые IP/CIDR (повторяемый) | Все IP |
| `--auto-close` | | Закрытие при простое (1m–24h) | Нет |
| `--max-lifetime` | | Макс. время жизни (1m–7d) | Нет |
| `--preset` | | Пресет безопасности | Нет |

## TCP-туннели

```bash
fxtunnel tcp <порт> [флаги]
```

### Автоматический порт

```bash
fxtunnel tcp 22
# → fxtun.dev:51234 (порт назначен сервером)
```

### Конкретный порт

```bash
fxtunnel tcp 22 --remote-port 2222
# → fxtun.dev:2222
```

### Примеры использования

SSH-доступ:
```bash
fxtunnel tcp 22 --remote-port 2222
# Подключение: ssh user@fxtun.dev -p 2222
```

PostgreSQL:
```bash
fxtunnel tcp 5432 --remote-port 15432 --allow-ip 10.0.0.0/8
# Подключение: psql -h fxtun.dev -p 15432 -U myuser mydb
```

### Заблокированные порты

На бесплатном плане некоторые порты недоступны для TCP-туннелей:

| Порт | Сервис |
|------|--------|
| 22 | SSH |
| 25 | SMTP |
| 53 | DNS |
| 135 | MSRPC |
| 139 | NetBIOS |
| 445 | SMB |
| 3306 | MySQL |
| 5432 | PostgreSQL |
| 6379 | Redis |
| 11211 | Memcached |
| 27017 | MongoDB |

Эти ограничения распространяются на **удалённый** порт. На платных планах ограничений нет.

### Все флаги TCP

| Флаг | Короткий | Описание | По умолчанию |
|------|----------|----------|--------------|
| `--remote-port` | `-r` | Удалённый порт (0 = авто) | 0 |
| `--allow-ip` | | Разрешённые IP/CIDR (повторяемый) | Все IP |
| `--auto-close` | | Закрытие при простое (1m–24h) | Нет |
| `--max-lifetime` | | Макс. время жизни (1m–7d) | Нет |

## UDP-туннели

```bash
fxtunnel udp <порт> [флаги]
```

Синтаксис и флаги аналогичны TCP. Используется для DNS, VoIP, игровых протоколов и других UDP-сервисов.

```bash
# DNS-сервер
fxtunnel udp 53 --remote-port 5353

# Игровой сервер
fxtunnel udp 27015 --auto-close 2h
```

## Управление поддоменами

### Просмотр зарезервированных поддоменов

```bash
fxtunnel domains list
```

```
Reserved domains (2/10):
  myapp        https://myapp.fxtun.dev    (created 2026-01-15)
  api          https://api.fxtun.dev      (created 2026-02-10)
```

### Резервирование поддомена

```bash
fxtunnel domains add myapp
# → Reserved: myapp → https://myapp.fxtun.dev
```

Зарезервированный поддомен закреплён за вашим аккаунтом — никто другой не сможет его занять.

### Проверка доступности

```bash
fxtunnel domains check myapp
```

### Освобождение поддомена

```bash
fxtunnel domains remove myapp
```

### Правила именования

- Длина: 3–32 символа
- Допустимые символы: `a-z`, `0-9`, `-` (дефис)
- Должен начинаться и заканчиваться буквой или цифрой
- Регистронезависимый (приводится к нижнему регистру)

### Зарезервированные имена

Следующие поддомены зарезервированы системой и не могут быть заняты:

`api`, `www`, `admin`, `mail`, `smtp`, `imap`, `pop`, `ftp`, `ns1`–`ns4`, `autoconfig`, `autodiscover`, `_dmarc`, `status`, `metrics`, `grafana`

## Пользовательские домены

Вы можете привязать собственный домен к туннелю.

### Добавление домена

```bash
fxtunnel domains custom add mydomain.com --target myapp
```

`--target` — поддомен, на который будет направлен трафик. Поддомен должен быть предварительно зарезервирован.

### Настройка DNS

Создайте DNS-запись, указывающую на fxTunnel:

**Для субдомена** (например, `tunnel.mydomain.com`):
```
CNAME  tunnel.mydomain.com  →  myapp.fxtun.dev
```

**Для корневого домена** (например, `mydomain.com`):
```
A  mydomain.com  →  IP-адрес сервера fxTunnel
```

### Проверка DNS

```bash
fxtunnel domains custom verify mydomain.com
```

После успешной верификации TLS-сертификат выпускается автоматически.

### Список и удаление

```bash
fxtunnel domains custom list
fxtunnel domains custom remove mydomain.com
```

## Конфигурационный файл

Для работы с несколькими туннелями используйте конфигурационный файл.

### Создание через мастер

```bash
fxtunnel init
```

Интерактивный мастер поможет создать `fxtunnel.yaml` в текущей директории.

### Полный пример

```yaml
# fxtunnel.yaml

server:
  address: "fxtun.dev:4443"       # Адрес сервера
  token: "sk_ваш_токен"           # API-токен
  insecure: false                  # Небезопасное TLS-соединение
  tls_verify: true                 # Проверять сертификат сервера
  compression: true                # Сжатие zstd

tunnels:
  - name: "webapp"                 # Имя туннеля (для логов)
    type: "http"                   # Тип: http, tcp, udp
    local_port: 3000               # Локальный порт
    subdomain: "myapp"             # Поддомен (только для HTTP)
    basic_auth: "user:Password123" # Basic Auth (только для HTTP)
    allow_ips:                     # Ограничение по IP
      - "10.0.0.0/8"
    auto_close: "1h"              # Закрытие при простое
    max_lifetime: "8h"            # Макс. время жизни

  - name: "ssh"
    type: "tcp"
    local_port: 22
    remote_port: 2222              # Удалённый порт (для TCP/UDP)

  - name: "dns"
    type: "udp"
    local_port: 53
    remote_port: 5353

reconnect:
  enabled: true                    # Автоматическое переподключение
  interval: 5s                     # Начальный интервал
  max_attempts: 0                  # 0 = бесконечно

inspect:
  enabled: true                    # Включить инспектор
  addr: "127.0.0.1:4040"          # Адрес инспектора
  max_entries: 1000                # Макс. кол-во записей
  max_body_size: 262144            # Макс. размер тела (256 КБ)

logging:
  level: "info"                    # debug, info, warn, error
  format: "console"                # console, json
```

### Запуск с конфигом

```bash
# Автоматический поиск fxtunnel.yaml в текущей директории
fxtunnel

# Или указать путь явно
fxtunnel --config path/to/config.yaml
```

### Приоритет настроек

Настройки применяются в порядке возрастания приоритета:

1. **Значения по умолчанию** (в коде)
2. **Конфигурационный файл** (fxtunnel.yaml)
3. **Переменные окружения** (FXTUNNEL_*)
4. **Флаги CLI** (--token, --server, ...)

### Поиск конфигурационного файла

Если `--config` не указан, клиент ищет файлы в таком порядке:

1. `fxtunnel.yaml` в текущей директории
2. `client.yaml` в текущей директории
3. `client.yaml` в `./configs/`
4. `~/.fxtunnel/client.yaml`

### Переменные окружения

Все параметры конфигурации можно задать через переменные окружения с префиксом `FXTUNNEL_`:

```bash
export FXTUNNEL_SERVER_ADDRESS="fxtun.dev:4443"
export FXTUNNEL_SERVER_TOKEN="sk_ваш_токен"
export FXTUNNEL_SERVER_COMPRESSION="false"
export FXTUNNEL_RECONNECT_INTERVAL="10s"
export FXTUNNEL_RECONNECT_MAX_ATTEMPTS="5"
export FXTUNNEL_INSPECT_ENABLED="true"
export FXTUNNEL_INSPECT_ADDR="127.0.0.1:4041"
export FXTUNNEL_LOGGING_LEVEL="debug"
export FXTUNNEL_LOGGING_FORMAT="json"
```

## Daemon-режим

Daemon-режим позволяет запускать туннели в фоне и управлять ими отдельно.

### Запуск

```bash
# В фоне (отсоединяется от терминала)
fxtunnel up

# В переднем плане (полезно для systemd или отладки)
fxtunnel up --foreground
```

Daemon читает конфигурацию из `fxtunnel.yaml` (или другого файла конфигурации) и открывает все описанные туннели.

### Статус

```bash
fxtunnel status
```

```
Daemon running (PID 12345)
Server: fxtun.dev:4443
  HTTP: https://myapp.fxtun.dev → localhost:3000
  TCP: fxtun.dev:2222 → localhost:22
  Uptime: 2h 15m
```

### Остановка

```bash
fxtunnel down
```

## Инспектор трафика

Инспектор перехватывает и отображает HTTP-трафик, проходящий через туннели. Полезен для отладки API, тестирования вебхуков, анализа запросов.

### Веб-интерфейс

По умолчанию инспектор доступен по адресу:

```
http://127.0.0.1:4040
```

Откройте в браузере для просмотра запросов в реальном времени.

Если порт 4040 занят, инспектор попробует порты 4041–4049.

### Отключение инспектора

```bash
fxtunnel http 3000 --no-inspect
```

Или в конфиге:

```yaml
inspect:
  enabled: false
```

### REST API инспектора

Инспектор предоставляет REST API для программного доступа:

#### Статус

```bash
curl http://127.0.0.1:4040/api/status
```

#### Список запросов

```bash
curl http://127.0.0.1:4040/api/requests/http
```

Параметры фильтрации:
- `tunnel_id` — фильтр по ID туннеля
- `method` — HTTP-метод (GET, POST, ...)
- `status` — код ответа (200, 404, ...)
- `search` — полнотекстовый поиск
- `limit` — макс. кол-во результатов (по умолчанию 100)
- `offset` — сдвиг для пагинации

#### Детали запроса

```bash
curl http://127.0.0.1:4040/api/requests/http/{id}
```

Возвращает полную информацию: заголовки запроса/ответа, тело, время выполнения.

#### Повтор запроса (Replay)

```bash
curl -X POST http://127.0.0.1:4040/api/requests/http \
  -H "Content-Type: application/json" \
  -d '{"id": "request-uuid"}'
```

Повторно отправляет запрос через туннель. Результат сохраняется как новая запись.

#### Потоковое отслеживание (SSE)

```bash
curl http://127.0.0.1:4040/api/requests/http/stream
```

Server-Sent Events стрим — получайте новые запросы в реальном времени:

```
data: {"id":"...","method":"GET","path":"/api/users","status":200,...}

data: {"id":"...","method":"POST","path":"/webhook","status":201,...}
```

#### Удаление записей

```bash
curl -X DELETE "http://127.0.0.1:4040/api/requests/http?tunnel_id=xxx"
```

#### Список активных туннелей

```bash
curl http://127.0.0.1:4040/api/tunnels
```

### Настройки инспектора

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `inspect.enabled` | Включить/выключить | `true` |
| `inspect.addr` | Адрес и порт | `127.0.0.1:4040` |
| `inspect.max_entries` | Макс. записей в буфере | `1000` |
| `inspect.max_body_size` | Макс. размер тела запроса/ответа | `262144` (256 КБ) |

## Предупредительная страница

При первом посещении HTTP-туннеля через браузер отображается предупредительная страница (interstitial). Она предупреждает, что контент предоставлен третьей стороной и не проверен — защита от фишинга.

### Когда показывается

Страница появляется если **все** условия выполнены:
- Метод запроса — `GET`
- Ответ имеет Content-Type `text/html`
- Используется поддомен (не пользовательский домен)
- Пользователь не является администратором

### Когда НЕ показывается

- **Пользовательские домены** — на кастомных доменах предупреждение не показывается
- **POST/PUT/DELETE и другие методы** — только GET вызывает interstitial
- **API-запросы** — JSON, XML и другие не-HTML ответы проходят без предупреждения
- **Cookie** — после нажатия «Продолжить» устанавливается cookie `_fxt_consent_<subdomain>` на 12 часов
- **Заголовок `X-FxTunnel-Skip-Warning`** — пропускает предупреждение (см. ниже)
- **Админ-аккаунт** — администраторы не видят предупреждений

### Пропуск через заголовок

Добавьте заголовок `X-FxTunnel-Skip-Warning` с любым непустым значением:

```bash
curl -H "X-FxTunnel-Skip-Warning: true" https://myapp.fxtun.dev
```

Это полезно для:
- Автоматизированных скриптов и CI/CD
- Тестирования вебхуков (Stripe, GitHub, Telegram и др.)
- Мониторинга и health-check'ов
- Любых программных обращений, где interstitial мешает

### Пропуск через cookie

Если вы заходите через браузер, нажмите кнопку «Продолжить» — cookie действует 12 часов.

## HTTP-заголовки

### Заголовки, устанавливаемые сервером

| Заголовок | Описание |
|-----------|----------|
| `X-Trace-Id` | Уникальный ID запроса (16 hex-символов) для отладки |
| `X-Forwarded-For` | IP-адрес клиента |
| `X-Forwarded-Proto` | Протокол (`http`) |
| `X-Forwarded-Host` | Оригинальный Host-заголовок |

Сервер **удаляет** входящие `X-Forwarded-*` заголовки и устанавливает свои — это защита от подделки.

### Заголовки для клиента

| Заголовок | Описание |
|-----------|----------|
| `X-FxTunnel-Skip-Warning: true` | Пропустить предупредительную страницу |

### Внутренние заголовки

| Заголовок | Описание |
|-----------|----------|

## Переподключение

При потере соединения клиент автоматически переподключается.

### Поведение

1. Соединение разрывается
2. Ожидание `interval` (по умолчанию 5 секунд)
3. Попытка переподключения
4. При неудаче — экспоненциальный backoff: `interval × 1.5`, максимум 2 минуты
5. Повтор до `max_attempts` (0 = бесконечно)

### Настройка

```yaml
reconnect:
  enabled: true       # Включить автопереподключение
  interval: 5s        # Начальный интервал
  max_attempts: 0     # 0 = бесконечно
```

### Что происходит после переподключения

- Туннели пересоздаются с той же конфигурацией
- Поддомены и порты сохраняются (если зарезервированы)
- Статистика (байты отправлено/получено) сбрасывается
- Таймеры `auto-close` и `max-lifetime` перезапускаются

Пресеты — готовые наборы настроек для типичных сценариев.

```bash
fxtunnel presets
```

### openclaw

Быстрое безопасное расшаривание с автогенерацией Basic Auth:

```bash
fxtunnel http 3000 --preset openclaw
```

```
Preset 'openclaw' credentials:
  Username: admin
  Password: x7kP9qM2lL5nR8sT

Tunnel established!
HTTP: https://ab1cd2ef.fxtun.dev
```

При каждом запуске генерируется новый случайный пароль (16 символов). Если вы явно укажете `--auth`, ваши значения перезапишут пресет.

## Лимиты и ограничения

### Лимиты по плану

| Параметр | Free | Base ($5) | Pro ($10) | Business ($15) |
|----------|------|-----------|-----------|----------------|
| Туннели | 3 | 5 | 15 | 50 |
| Зарезервированные поддомены | 0 | 5 | 15 | 50 |
| Пользовательские домены | 0 | 1 | 5 | 50 |
| API-токены | 1 | 5 | 10 | 50 |
| Инспектор | Нет | Безлимит | Безлимит | Безлимит |

### Rate limiting

| Протокол | Лимит по умолчанию |
|----------|-------------------|
| HTTP | 3600 запросов/мин на туннель |
| TCP | 1800 соединений/мин на туннель |
| UDP | 10000 пакетов/сек на туннель |

Дополнительно действует лимит на IP-адрес — 1/10 от лимита туннеля.

При превышении лимита HTTP-запросы получают ответ `429 Too Many Requests`.

### Размер тела в инспекторе

По умолчанию инспектор сохраняет первые 256 КБ тела запроса/ответа. Настраивается через `inspect.max_body_size`.

## FAQ

### Как получить постоянный URL?

Зарезервируйте поддомен:

```bash
fxtunnel domains add myapp
fxtunnel http 3000 --domain myapp
# Всегда https://myapp.fxtun.dev
```

Без резервирования поддомен может занять другой пользователь.

### Как дать доступ коллеге?

Вариант 1 — Basic Auth:
```bash
fxtunnel http 3000 --domain demo --auth team:SharedPass123
# Поделитесь URL и паролем
```

Вариант 2 — Ограничение по IP:
```bash
fxtunnel http 3000 --allow-ip 203.0.113.10
```

Вариант 3 — Пресет:
```bash
fxtunnel http 3000 --preset openclaw
# Отправьте сгенерированные логин/пароль
```

### Как использовать с Docker?

Укажите адрес хоста вместо localhost:

```bash
# Docker контейнер слушает на порту 8080
fxtunnel http 8080
```

Если контейнер слушает на другом сетевом интерфейсе:

```yaml
tunnels:
  - name: "docker-app"
    type: "http"
    local_addr: "172.17.0.2"    # IP контейнера
    local_port: 8080
    subdomain: "myapp"
```

### Как отладить вебхуки?

1. Запустите туннель с инспектором:
```bash
fxtunnel http 3000 --domain webhooks
```

2. Укажите URL `https://webhooks.fxtun.dev/webhook` в настройках сервиса (Stripe, GitHub, Telegram и др.)

3. Откройте инспектор `http://127.0.0.1:4040` — все входящие вебхуки отобразятся в реальном времени

4. Используйте Replay для повторной отправки:
```bash
curl -X POST http://127.0.0.1:4040/api/requests/http \
  -H "Content-Type: application/json" \
  -d '{"id": "request-id-from-inspector"}'
```

### Как проксировать WebSocket?

WebSocket-соединения проксируются автоматически — никакой дополнительной настройки не требуется:

```bash
fxtunnel http 3000 --domain ws-app
# ws://ws-app.fxtun.dev и wss://ws-app.fxtun.dev работают из коробки
```

### Как запускать при старте системы?

Создайте systemd-сервис:

```ini
# /etc/systemd/system/fxtunnel.service
[Unit]
Description=fxTunnel Client
After=network.target

[Service]
Type=simple
ExecStart=/home/user/.local/bin/fxtunnel up --foreground
Restart=always
RestartSec=5
User=user
WorkingDirectory=/home/user/project

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable --now fxtunnel
```

### Инспектор недоступен?

Убедитесь, что план поддерживает инспектор (доступен на платных планах). На бесплатном плане инспектор отключён на стороне сервера.

Если порт 4040 занят, проверьте вывод при запуске — клиент укажет фактический порт (4041–4049).

### Ошибка «port is blocked for security reasons»?

Некоторые TCP-порты заблокированы на бесплатном плане (SSH, базы данных и др.). Перейдите на платный план или используйте другой удалённый порт.
