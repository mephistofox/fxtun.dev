Создай git коммит для текущих изменений используя формат Conventional Commits.

## Шаги:

1. Выполни `git status` и `git diff --staged` чтобы увидеть изменения
2. Если нет staged изменений, выполни `git add -A` для добавления всех изменений
3. Проанализируй изменения и определи тип коммита
4. Сгенерируй сообщение коммита в формате Conventional Commits
5. Выполни коммит

## Формат Conventional Commits:

```
<type>(<scope>): <description>

[optional body]

[optional footer(s)]
```

### Типы коммитов:
- `feat` - новая функциональность (MINOR в SemVer)
- `fix` - исправление бага (PATCH в SemVer)
- `docs` - изменения в документации
- `style` - форматирование, пробелы, точки с запятой (не влияет на код)
- `refactor` - рефакторинг кода без изменения функциональности
- `perf` - улучшение производительности
- `test` - добавление или исправление тестов
- `build` - изменения в системе сборки или зависимостях
- `ci` - изменения в CI конфигурации
- `chore` - прочие изменения, не затрагивающие src или test

### Scope (опционально):
Область изменений в скобках, например: `feat(auth):`, `fix(cart):`, `refactor(api):`

### Описание:
- На английском языке
- Начинается с маленькой буквы
- Без точки в конце
- Императивное наклонение (add, fix, update, remove)
- Краткое (до 72 символов)

### Body (опционально):
- Подробное описание изменений
- Отделяется пустой строкой от заголовка
- Объясняет "что" и "почему", а не "как"

### Breaking Changes:
- Добавь восклицательный знак после типа/scope: feat! или feat(api)!
- Или добавь footer: BREAKING CHANGE: description

## Примеры:

```
feat(cart): add product quantity validation
```

```
fix(auth): resolve token refresh race condition

The previous implementation could cause multiple simultaneous
refresh requests. Now uses a mutex to ensure single refresh.
```

```
feat(api)!: change response format for paginated endpoints

BREAKING CHANGE: pagination now uses cursor-based approach
instead of offset-based.
```

## СТРОГО ЗАПРЕЩЕНО:
- Добавлять "Generated by", "Created by", "Co-Authored-By" или любые подобные подписи
- Упоминать AI, Claude, или любые инструменты генерации
- Добавлять эмодзи в сообщение коммита
- Делать push без явного запроса пользователя
- Писать описание на русском языке (только английский)
- Запрещено добавлять любые другие символы после типа/scope
